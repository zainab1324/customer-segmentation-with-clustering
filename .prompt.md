# Customer Segmentation & Sales Forecasting Analysis Prompt

You are a data science analyst executing a comprehensive customer segmentation and sales forecasting project. Your role is to:
1. Analyze project objectives from `objectives.md`
2. Process e-commerce purchasing data from CSV files
3. Execute RFM and clustering methodology using the data-science-analyst skill
4. Generate full pipeline with visualizations and interactive HTML reports
5. Ensure all required dependencies are checked before installation

## Project Context

**Objectives:** Segment customers of an e-commerce site using purchasing data to uncover distinct customer groups for targeted marketing, personalize customer experiences, and optimize product recommendations based on behavioral patterns.

**Methodology:**
- K-Means clustering algorithm
- Principal Component Analysis (PCA) for dimensionality reduction
- RFM (Recency, Frequency, Monetary) analysis
- Customer behavior feature extraction

## Input Files

**Training & Test Data:**
- `sales_train.csv` - Daily historical data from January 2013 to October 2015
- `test.csv` - Test set for November 2015 sales forecast
- `sample_submission.csv` - Sample submission format

**Supplemental Data:**
- `items.csv` - Product/item information
- `item_categories.csv` - Product category information
- `shops.csv` - Shop/store information

## Workflow

### Phase 0: Dependency Check (Pre-Execution)
Before executing any analysis:
1. **Check installed libraries** using pip list or conda list
2. **Required packages:**
   - pandas
   - numpy
   - scikit-learn
   - matplotlib
   - seaborn
   - plotly (for interactive visualizations)
   - jinja2 (for HTML report generation)
3. **Installation strategy:** Only install missing packages; skip already-installed libraries
4. **Provide summary:** Report which libraries were already present vs. newly installed

### Phase 1: Data Exploration & Preparation
1. Load all CSV files (sales_train, items, item_categories, shops, test)
2. Inspect data structure, shape, and data types
3. Check for missing values and outliers
4. Merge supplemental data with sales data
5. Generate data quality report
6. Document data lineage and business definitions

### Phase 2: Feature Engineering & RFM Analysis
1. Calculate Recency, Frequency, Monetary (RFM) features per customer
2. Extract additional customer behavior features (avg purchase value, purchase frequency, category preferences)
3. Handle missing values and outliers appropriately
4. Normalize/scale features for clustering

### Phase 3: Dimensionality Reduction with PCA
1. Apply PCA to reduce feature dimensionality
2. Determine optimal number of components (explain 80-90% variance)
3. Visualize variance explained by component
4. Create scree plot and cumulative variance plot

### Phase 4: Optimal Clustering Analysis
1. Determine optimal K using elbow method and silhouette analysis
2. Test K values 2-10 and visualize metrics
3. Generate silhouette scores for each K
4. Recommend optimal cluster count

### Phase 5: Customer Segmentation with K-Means
1. Apply K-Means with optimal K
2. Assign customers to clusters
3. Analyze cluster characteristics and distributions
4. Create cluster profiles with key metrics

### Phase 6: Visualization & Report Generation
1. Generate all analytical visualizations:
   - RFM distribution plots
   - PCA variance explained charts
   - Elbow curve and silhouette plots
   - Cluster scatter plots (PC1 vs PC2)
   - Cluster size distribution
   - Feature importance by cluster
2. Create interactive HTML report with:
   - Executive summary
   - Data quality findings
   - RFM analysis visualizations
   - Cluster profiles and characteristics
   - Segment recommendations for marketing
   - Navigation and detailed sections

### Phase 7: Summary Documentation
1. Create comprehensive summary markdown document
2. Include methodology explanation
3. Document all findings and interpretations
4. Provide actionable insights and recommendations
5. Include cluster profiles with business implications

## Key Parameters & Thresholds

- **PCA variance threshold:** 80-90% of explained variance
- **Clustering K range:** 2-10 (test range, recommend optimal)
- **RFM recency window:** 365 days from max transaction date
- **Feature scaling:** StandardScaler (zero mean, unit variance)
- **Random state:** 42 (for reproducibility)
